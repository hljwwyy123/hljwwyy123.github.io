## 从 Vibe Coding 到 SDD：一部正在发生的技术思想史
**——当 AI 开始写软件，我们在寻找一种新的“文明秩序”**
有时候我会想，如果未来的技术哲学家回望 2023–2026 这几年，
他们大概会像刘擎解读现代性那样，
把这段时间称作*“AI 软件文明的摇摆时期”：*
旧模式瓦解，新范式未立；
自由让人兴奋，失序让人恐惧。

而我们这些程序员，恰好生活在断裂的缝隙中。

第一次使用 Cursor 或 Copilot 的那一刻，你可能和我一样，心里闪过过一个念头：

> “原来写代码也可以这么轻盈、这么流畅，几乎带着一种诗意。”

那是 **vibe coding 的黄金时代。**
你给几句提示，它就开始“悟”，开始“写”，
像一位无眠的天才伙伴，陪你通宵 jam 代码。

但就在那种轻松让人沉醉的时候，我们忽略了一件事：
**魔法从来不会无代价。**
### 一、Vibe Coding：一个浪漫而危险的“魔法时代”

回想那段时间，我们以为 AI 是一个“理解你”的搭档。
你一句“帮我写一个登录功能”，它就立刻开写。
有点像鲁迅说的“我只消动动嘴皮”，梦想就会喷涌出来。

但很快我们发现——
这段恋爱并不稳定。

同样的 prompt，给出不一样的代码；
修一个 bug，引出三个新的；
你越依赖，它越发散；
你越催促，它越幻觉。

那种体验，很像塞缪尔·贝克特写的那句：“继续失败，继续尝试，失败得更好。”

Vibe coding 就是这样一个时代：
轻盈、即兴、迷人，但也脆弱、混乱、无法复现。

我们都醉过。
也都崩溃过。

### 二、6A 工作流：秩序从浪漫中长出来

当 AI 能完成的任务越来越大，一件事变得清晰：

如果没有流程，AI 就只是一台“猜测机器”。

于是 6A 工作流出现了。

它不是一个工具，而是一种心态变化：
从“让 AI 帮我写代码” → “我来教 AI 如何做工程”。

6A 六个阶段非常像工程学的屏障系统（Barrier System）——
层层过滤，避免灾难：

Align（对齐）：需求必须清晰，不准“我猜你大概想要…”。

Architect（架构）：先设计再写，不允许边走边改。

Atomize（原子化）：大任务要拆开，直到 AI 不可能误解。

Approve（审批）：人类必须审阅，中间不能跳步骤。

Automate（执行）：按文档执行，不许自由发挥。

Assess（评估）：必须验收，不合格就打回重做。

如果 vibe coding 是“即兴爵士乐”，
6A 就是开始读谱、开始排练、开始正式演出。

它让 AI 不敢偷懒，也无法乱跑。

但 6A 的本质，是一个“准工程化”的体系：
流程开始出现，标准开始出现，但仍然依赖人类大量监管。

文明开始形成，但还不稳固。

### 三、大厂内部的启示：从阿里到腾讯，每家都在挣扎同样的问题

其实，AI 工程化不是个人问题，而是组织问题。

你用过阿里的 Qrder / OrderFlow，
也体验过腾讯的 TCB AI。

你会发现两者虽然形式不同，但实质相同：

AI 不能直接写代码，必须经过规格化的描述

所有步骤都有可追踪的“工单逻辑”

有明确的审批机制

有校验、有回滚、有对齐，有边界

AI 的角色从“创作者”变成“执行器”

大厂比个人更早意识到一个残酷事实：

“AI + vibe” 写的是玩具，
“AI + process” 才可能写的是系统。

这也解释了为什么所有公司——阿里、腾讯、微软、谷歌、OpenAI、Cursor——
在过去一年几乎同步走向同一个方向：

从 GPT 的魔法 → Spec 的工程。

### 四、SDD：当 AI 开始遵守“法典”

如果 6A 是流程的萌芽，
SDD（Spec Driven Development）就是法典的降临。

它解决的是 vibe coding 和 6A 都解决不了的问题：

需求模糊

类型不清晰

状态机混乱

边界漂移

AI 发散

模块不一致

协作无法复现

Spec 出现后，人类第一次能够以一种“可执行语言”跟 AI 沟通：

什么是允许的

什么是不允许的

状态有哪些

API 的输入输出是什么

哪些边界不能跨

哪些错误必须处理

它不像法律那样生硬，也不像自然语言那样随意。
它是一个“半形式化的现实世界模型”。

AI 在 spec 面前，像在阅读一张施工图纸：
不需要猜，也不能乱改。

它在这里被“文明化”了。
变得可靠、可控、可复现。

SDD 是 AI 工程真正的转折点。

### 五、为什么 Vibe → 6A → SDD 是不可逆的？

有人问我：“能不能只用 vibe coding？我喜欢那种自由。”

就像有人问：
“能不能一直住在棚户区？那里便宜，又充满人情味。”

但城市的发展有自己的方向。

① 工具越强，越需要结构

GPT-5 比 GPT-4 聪明太多，它能创造更多，也能更高速地幻觉。
能力越强，越需要规训。

② 项目越复杂，越依赖文档

没有 spec 的 AI 工程，就像没有图纸的施工现场。
它不是创新，是赌博。

③ 多人协作必须去语义化

自然语言再清晰，也会模糊。
Spec 是 AI 的共同语言，是协作的基石。

④ 可复现性是文明的核心

vibe coding 是体验；
SDD 是生产力。

### 六、真正的未来：当 spec 成为新的“世界模型”

这篇文章不是讲一个更好的“工作流”。
它讲的是一个更深的变化：

未来的软件，不是由工程师写的，
而是由 spec → agent → simulation 链条生成的。

Spec 不是文档，
它是“世界的定义”。

写 spec 就像在写一段文明的法律、地图和物理规则。

AI 在 spec 上构建的系统，
就像在一个稳定的宇宙里点亮新的星群。

这个未来比 vibe coding 更宏大，
也比 6A 工作流更自由——
因为自由不来自混乱，而来自秩序之后的再创造。

### 七、尾声：我们正在见证一种新的“工程哲学”

从 vibe coding 的浪漫，
到 6A 的秩序，
到 SDD 的法典化文明——
这是 AI 时代程序员共同经历的“现代性之旅”。

它像刘擎理解现代世界那样：
自由令人兴奋，秩序令人压抑，但离了谁都不行。

程序员的铁笼既是禁锢，也是庇护。
SDD 是那种笼子，但也只有在这样的笼子里，
AI 写出的软件才能稳固地站在现实里。

我们写下的每一条 spec，
都像是在一片混沌的数字世界里铺下一块可靠的地基。

未来的系统会在其上长成森林。
我们会回头想起 vibe coding 的时代，
像回忆一段无忧无虑的初恋——
迷人、轻盈，却无法建造房屋。

文明的下一步，从这里开始。
